# Projekt: Minigame (Coding-Challenge)

Diese Regeln gelten für jede Session in diesem Projekt und sind verbindlich.

## Git-Workflow (verbindlich)

1. **Push nach jedem Arbeitsschritt.**
   Sobald ein Stück Arbeit fertig ist, sofort committen und pushen:
   ```
   git add -A
   git commit -m "<kurze, klare Message>"
   git push
   ```
   Sehr regelmäßig, an den Fortschritt gekoppelt — kein Timer. Immer wenn wieder
   ein Teil fertig ist, wird gepusht.

2. **Nur zu `origin` pushen.**
   `origin` ist mein eigenes Repo: `https://github.com/Marvinrw3/minigame.git`.
   Das Push-Ziel wird niemals geändert und es wird niemals woanders hingepusht.

3. **Session-Start: erst orientieren.**
   Zu Beginn jeder neuen Session zuerst `git log` und `git status` anschauen,
   kurz orientieren, dann nahtlos weiterbauen — weiterhin mit Push nach jedem
   Schritt.

**Ziel:** Der komplette Fortschritt landet fortlaufend und nachvollziehbar in
meinem eigenen Repo.
